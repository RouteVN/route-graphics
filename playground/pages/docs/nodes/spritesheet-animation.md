`spritesheet-animation` renders frame animation from a spritesheet image plus inline atlas metadata.

`atlas` is usually generated by external tools, not written by hand.

Try it in the [Playground](/playground/?template=spritesheet-animation-demo).

## Used In

- `elements[]`

## Field Reference

| Field      | Type   | Required | Default | Notes                              |
| ---------- | ------ | -------- | ------- | ---------------------------------- |
| `id`       | string | Yes      | -       | Element id.                        |
| `type`     | string | Yes      | -       | Must be `spritesheet-animation`.   |
| `x`        | number | Yes      | -       | Position before anchor transform.  |
| `y`        | number | Yes      | -       | Position before anchor transform.  |
| `width`    | number | Yes      | -       | Render width.                      |
| `height`   | number | Yes      | -       | Render height.                     |
| `src`      | string | Yes      | `""`    | Texture alias/URL for sheet image. |
| `atlas`    | object | Yes      | -       | Inline atlas metadata.             |
| `clips`    | object | No       | `{}`    | Optional named frame sequences.    |
| `playback` | object | Yes      | -       | Playback settings (see below).     |
| `anchorX`  | number | No       | `0`     | Anchor offset ratio.               |
| `anchorY`  | number | No       | `0`     | Anchor offset ratio.               |
| `alpha`    | number | No       | `1`     | Opacity `0..1`.                    |
| `blur`     | object | No       | -       | Directional Gaussian blur.         |

## Blur

`blur` requires explicit horizontal and vertical axes. There is no scalar shorthand.

| Field              | Type    | Required | Default | Notes                                             |
| ------------------ | ------- | -------- | ------- | ------------------------------------------------- |
| `x`                | number  | Yes      | -       | Horizontal blur strength in pixels.               |
| `y`                | number  | Yes      | -       | Vertical blur strength in pixels.                 |
| `quality`          | number  | No       | `4`     | Number of blur passes. Higher is smoother/slower. |
| `kernelSize`       | number  | No       | `5`     | One of `5`, `7`, `9`, `11`, `13`, `15`.           |
| `repeatEdgePixels` | boolean | No       | `false` | Clamp edge pixels instead of padding blur bounds. |

### `atlas`

```yaml
atlas:
  frames:
    idle-0: { x: 0, y: 0, width: 64, height: 64 }
    idle-1: { x: 64, y: 0, width: 64, height: 64 }
  width: 128
  height: 64
  scale: 1
```

For simple sheets, you can use the short frame form above.

Route Graphics also accepts common generated atlas JSON formats:

- PixiJS `SpriteSheetJson`
- TexturePacker `JSON Hash`
- TexturePacker `JSON Array`
- Phaser atlas `JSON Hash`
- Phaser atlas `JSON Array`
- Aseprite `json-hash`
- Aseprite `json-array`

Generated atlases may include fields such as `rotated`, `trimmed`, `spriteSourceSize`, `sourceSize`, `anchor`, `pivot`, and `borders`. These are preserved when relevant.

If the atlas JSON includes an `animations` object, Route Graphics treats those entries as named clips automatically.

If the atlas JSON includes Aseprite `meta.frameTags`, Route Graphics converts those tags into named clips automatically.

Aseprite per-frame `duration` values are accepted in the JSON, but Route Graphics does not currently import variable frame timing from them. Playback timing still comes from `playback.fps`.

Example generated frame entry:

```yaml
atlas:
  frames:
    rollSequence0001.png:
      frame: { x: 468, y: 2, w: 169, h: 226 }
      rotated: false
      trimmed: true
      spriteSourceSize: { x: 3, y: 4, w: 169, h: 226 }
      sourceSize: { w: 175, h: 240 }
  animations:
    roll:
      - rollSequence0000.png
      - rollSequence0001.png
```

Route Graphics normalizes all supported atlas inputs into one internal format before rendering.

### `playback`

| Field      | Type     | Required | Default | Notes                                          |
| ---------- | -------- | -------- | ------- | ---------------------------------------------- |
| `clip`     | string   | No       | -       | Named clip from `clips` or `atlas.animations`. |
| `frames`   | string[] | No       | `[]`    | Explicit frame-name sequence.                  |
| `fps`      | number   | No       | `30`    | Playback rate in frames per second.            |
| `loop`     | boolean  | No       | `true`  | Loop playback.                                 |
| `autoplay` | boolean  | No       | `true`  | Start playback automatically.                  |

## Behavior Notes

- No element-level interaction events are emitted.
- In debug mode, playback uses debug helpers instead of auto-play.
- Multi-atlas manifests are not supported. Use a single image plus a single atlas JSON object.

## Example: Minimal

```yaml
elements:
  - id: fighter
    type: spritesheet-animation
    x: 540
    y: 240
    width: 128
    height: 128
    src: fighter-spritesheet
    atlas:
      frames:
        frame-0: { x: 0, y: 0, width: 64, height: 64 }
        frame-1: { x: 64, y: 0, width: 64, height: 64 }
      width: 128
      height: 64
    playback:
      frames: [frame-0, frame-1]
```

## Example: Named Clip Playback

```yaml
elements:
  - id: coin-spin
    type: spritesheet-animation
    x: 980
    y: 160
    width: 96
    height: 96
    src: coin-sheet
    atlas:
      frames:
        spin-0: { x: 0, y: 0, width: 32, height: 32 }
        spin-1: { x: 32, y: 0, width: 32, height: 32 }
        spin-2: { x: 64, y: 0, width: 32, height: 32 }
      width: 96
      height: 32
    clips:
      spin: [spin-0, spin-1, spin-2]
    playback:
      clip: spin
      fps: 18
      loop: true
```

## Example: Spritesheet Animation With Enter Motion

```yaml
elements:
  - id: enemy
    type: spritesheet-animation
    x: 840
    y: 300
    width: 140
    height: 140
    src: enemy-sheet
    atlas:
      frames:
        idle-0: { x: 0, y: 0, width: 70, height: 70 }
        idle-1: { x: 70, y: 0, width: 70, height: 70 }
      width: 140
      height: 70
    clips:
      idle: [idle-0, idle-1]
    playback:
      clip: idle
      fps: 12

animations:
  - id: enemy-enter
    targetId: enemy
    type: live
    tween:
      x:
        initialValue: 1180
        keyframes:
          - value: 840
            duration: 500
            easing: linear
      alpha:
        initialValue: 0
        keyframes:
          - value: 1
            duration: 500
            easing: linear
```
